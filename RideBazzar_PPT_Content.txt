PAGE 8: DATABASE SCHEMA (ER DIAGRAM)
================================================================================
Title: Database Schema (ER Diagram)

CORE ENTITIES:

1. USERS TABLE
   - user_id (Primary Key)
   - username
   - email
   - password (hashed)
   - phone
   - role (admin, owner, customer)
   - profile_picture
   - created_at
   - updated_at

2. CARS TABLE
   - car_id (Primary Key)
   - owner_id (Foreign Key → USERS)
   - make
   - model
   - year
   - color
   - price_per_day
   - location
   - status (available, booked, maintenance)
   - description
   - created_at
   - updated_at

3. BOOKINGS TABLE
   - booking_id (Primary Key)
   - user_id (Foreign Key → USERS)
   - car_id (Foreign Key → CARS)
   - booking_date
   - start_date
   - end_date
   - status (pending, confirmed, completed, cancelled)
   - total_amount
   - booking_amount
   - created_at
   - updated_at

4. TRANSACTIONS TABLE
   - txn_id (Primary Key)
   - booking_id (Foreign Key → BOOKINGS)
   - user_id (Foreign Key → USERS)
   - amount
   - txn_date
   - payment_method
   - status (pending, success, failed, refunded)
   - transaction_id (gateway reference)
   - created_at

5. INQUIRIES TABLE
   - inquiry_id (Primary Key)
   - user_id (Foreign Key → USERS)
   - car_id (Foreign Key → CARS)
   - inquiry_date
   - message
   - status (open, resolved, closed)
   - response
   - created_at
   - updated_at

KEY RELATIONSHIPS:
• USERS POSTS CARS (1:N) - One user can post multiple cars
• CARS CONTAINED_IN BOOKINGS (1:N) - One car can have multiple bookings
• USERS BOOK CARS (1:N via Bookings) - Users make multiple bookings
• BOOKINGS PROCESSES TRANSACTIONS (N:1) - Multiple payments per booking
• USERS RECEIVE INQUIRIES (1:N) - Users receive multiple inquiries
