📋 Bookings Endpoints

GET /api/bookings/my-bookings
Returns user's purchases as buyer
Response: { success: true, bookings: [...] }

GET /api/bookings/my-sales
Returns user's sales as seller
Response: { success: true, sales: [...] }

POST /api/bookings/
Create new booking
Body: { carId, bookingAmount, totalAmount, buyerInfo }

GET /api/bookings/:id
Get single booking details
Response: { success: true, data: {...} }

💬 Inquiries Endpoints

POST /api/inquiries/submit
Submit new inquiry
Body: { carId, message, phone }

GET /api/inquiries/received
Get inquiries received (as seller)
Response: { success: true, data: [...] }

GET /api/inquiries/sent
Get inquiries sent (as buyer)
Response: { success: true, data: [...] }

PUT /api/inquiries/:inquiryId/status
Update inquiry status
Body: { status, response }

💳 Payments Endpoints

POST /api/payments/process
Process payment
Body: { bookingId, amount, paymentMethod, paymentDetails }

GET /api/payments/history
Get user's transaction history
Response: { success: true, data: [...] }

GET /api/payments/admin/all
Get all transactions (admin only)
Response: { success: true, data: [...] }

📝 Implementation Notes

Database Tables:
• bookings - User purchases and sales

• transactions - Payment records
• cars - Available vehicles
• users - User accounts

Key Fields:
• booking.status: pending/confirmed/completed
• inquiry.status: pending/responded/closed
• transaction.payment_status: success/failed
